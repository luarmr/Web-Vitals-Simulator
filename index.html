How to Improve Web Vitals

  • Optimize and compress images

    Use modern image formats like WebP or AVIF which are 25-50% smaller than JPEG/PNG. Compress images without significant quality loss using tools like ImageOptim, TinyPNG, or Squoosh. Consider responsive images with the srcset attribute to serve different sized images for different devices.

  • Implement lazy loading for below-the-fold images

    Only load images when they're about to enter the viewport using the native loading="lazy" attribute or a JavaScript library like lozad.js. This prioritizes critical content and reduces initial page load time.

  • Remove render-blocking resources

    Use async or defer attributes on non-critical scripts. Move critical CSS inline and load non-critical CSS asynchronously. Consider using resource hints like preload, prefetch, and preconnect to prioritize critical resources.

  • Minimize critical CSS

    Extract and inline only the CSS needed for above-the-fold content. Use tools like Critical, PurgeCSS, or UnCSS to identify and extract essential styles, reducing the CSS blocking time.

  • Implement server-side rendering or static generation

    Pre-render pages on the server to send complete HTML to the browser, reducing client-side rendering time. Consider frameworks like Next.js or Gatsby that support SSR/SSG for improved performance.

  • Use a CDN to reduce network latency

    Deploy content across multiple geographic locations to reduce round-trip time to the server. CDNs like Cloudflare, Fastly, or Akamai can significantly reduce time to first byte and speed up resource delivery.

  • Eliminate render-blocking resources

    Move JavaScript and CSS that isn't necessary for the initial render to load asynchronously. Use async or defer for scripts and load non-critical CSS with JavaScript or media="print" with an onload handler to switch it to all.

  • Minify CSS

    Remove whitespace, comments, and unnecessary characters from CSS files to reduce file size. Use tools like CSSNano, CleanCSS, or build systems like Webpack with appropriate plugins to automatically minify during builds.

  • Remove unused CSS

    Eliminate CSS rules that aren't applied to any elements on your page. Tools like PurgeCSS, UnCSS, or Chrome DevTools Coverage tab can help identify unused CSS, potentially reducing CSS payload by 50% or more.

  • Preconnect to required origins

    Establish early connections to important third-party domains with <link rel="preconnect"> tags. This performs DNS lookups, TCP handshakes, and optional TLS negotiation in advance, saving 100-500ms when resources are actually requested.

  • Reduce server response times (TTFB)

    Optimize server processing by caching database queries, using efficient database indexing, implementing application-level caching, and optimizing server code. Consider using edge computing or serverless functions to reduce latency.

  • Avoid multiple page redirects

    Each redirect adds a complete roundtrip (HTTP request and response) before the browser can load the page. Eliminate unnecessary redirects, especially redirect chains, and use server-side redirection instead of client-side when possible.

  • Preload key requests

    Use <link rel="preload"> to inform the browser about critical resources needed for the current page. This is especially useful for fonts, critical JavaScript/CSS files, or important images that appear above the fold.

  • Break up long tasks (>50ms)

    Chunk JavaScript execution into smaller tasks using techniques like code-splitting, time-slicing, or requestIdleCallback. This prevents the main thread from being blocked for extended periods and helps maintain responsiveness during interactions.

  • Optimize your JavaScript (code splitting, tree shaking)

    Use modern bundling techniques to only load necessary code. Code splitting delivers JavaScript in smaller chunks, while tree shaking eliminates unused code. Modern bundlers like Webpack, Rollup, or esbuild can automate these optimizations.

  • Defer non-critical JavaScript

    Load and execute non-essential JavaScript after critical page content is interactive. This includes third-party scripts, analytics, and features not needed for the core user experience. Consider loading these scripts on user interaction or after the page is idle.

  • Use a web worker for heavy processing

    Move CPU-intensive tasks off the main thread to web workers. This includes data processing, complex calculations, and encoding/decoding operations, keeping the main thread free to handle user interactions.

  • Minimize main thread work

    Reduce work on the main thread by optimizing JavaScript execution, minimizing DOM manipulations, and avoiding layout thrashing. Use techniques like debouncing/throttling for event handlers and batching DOM updates to reduce browser reflow/repaint cycles.

  • Optimize event handlers

    Keep event handlers lightweight and efficient. Use event delegation instead of attaching many individual listeners. Remove unnecessary event listeners when they're no longer needed, and avoid expensive operations like DOM queries inside frequently triggered events.

  • Implement proper request prioritization

    Ensure critical resources are loaded with appropriate priority using resource hints like preload, preconnect, and prefetch. This helps browsers allocate bandwidth more efficiently, ensuring interaction-critical resources are available when needed.

  • Use requestAnimationFrame for visual updates

    Synchronize visual changes with the browser's rendering cycle using requestAnimationFrame(). This optimizes animations and visual updates by ensuring they occur at the right time in the rendering pipeline, reducing jank and improving responsiveness.

  • Set explicit width and height for images and videos

    Always specify dimensions for media elements to reserve space before they load. Use the width and height attributes or CSS properties to maintain the aspect ratio. Modern browsers use this information to calculate the correct space even before the image loads.

  • Reserve space for dynamic content

    Use placeholder elements with minimum heights or skeleton screens for content that loads asynchronously. This prevents layout shifts when the content eventually loads. For ads, embed containers with specific dimensions that match the expected ad size.

  • Avoid inserting content above existing content

    Adding content above what the user already sees (like banners, notifications, or lazy-loaded elements) causes everything below to shift down. Instead, reserve space for these elements from the start or insert them in a way that doesn't displace existing content.

  • Minimize font swaps by preloading fonts

    Font loading can cause text to reflow. Preload important fonts with <link rel="preload" as="font"> and use font-display: optional or font-display: swap with fallback fonts that closely match the final font's metrics to minimize shift.

  • Ensure ads have reserved space

    Specify fixed dimensions for ad containers that match the ad sizes you're serving. If ad sizes are variable, consider placing them in locations where shifts won't affect the main content, like sidebars or between distinct content sections.

  • Avoid animations that cause layout shifts

    Use CSS properties that only trigger compositing (transform, opacity) rather than those that cause layout recalculation (height, width, position). The former are handled by the GPU and don't cause layout shifts, making animations smoother and less disruptive.

  • Optimize server response time

    Improve backend performance by optimizing application code, using faster web servers, implementing efficient routing, and reducing middleware overhead. Consider upgrading server hardware or scaling horizontally for better performance under load.

  • Implement server-side caching

    Cache database queries, API responses, and rendered HTML templates to avoid regenerating content for every request. Use memory-based caching systems like Redis or Memcached for frequently accessed data to reduce processing time and database load.

  • Use a CDN

    Content Delivery Networks place your static assets and potentially your entire site on edge servers closer to your users. This dramatically reduces network latency and improves TTFB by 30-60% for users around the world by serving content from the nearest geographic location.

  • Optimize database queries

    Improve database performance by adding appropriate indexes, denormalizing data where necessary, optimizing query patterns, and using database query caching. Monitor slow queries and use database-specific optimization techniques for your chosen platform.

  • Implement HTTP/2 or HTTP/3

    Modern HTTP protocols improve connection efficiency with features like multiplexing, header compression, and server push. HTTP/3 uses QUIC instead of TCP, further reducing latency especially on mobile networks with frequently changing connections.

  • Reduce DNS lookup time with DNS prefetching

    Use <link rel="dns-prefetch"> for domains you'll connect to, reducing DNS resolution time by 50-300ms. This is especially important for third-party resources from multiple domains that are critical for your page rendering.
